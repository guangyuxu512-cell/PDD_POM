**ask 21 — 选择器与 POM 动作分离重构**

目标：新建 `selectors/` 模块，将 `pages/` 中所有硬编码的选择器提取到选择器文件，page 层只保留动作方法。

**新建文件：**

`selectors/__init__.py`

- 空文件

`selectors/基础页选择器.py`

- class 基础页选择器
- 通用弹窗关闭按钮: list[str]
- 从现有 `pages/基础页.py` 的 `关闭所有弹窗` 方法中提取选择器值，保留现有值

`selectors/登录页选择器.py`

- class 登录页选择器
- 从现有 `pages/登录页.py` 中提取所有选择器值，保留现有值（这些是已验证可用的）

`selectors/商品列表页选择器.py`

- class 商品列表页选择器
- 从现有 `pages/商品列表页.py` 中提取所有选择器，保留现有值
- 包含：搜索类型下拉、商品ID选项、商品ID输入框、查询按钮、发布相似品链接、发布相似品弹窗确认按钮

`selectors/发布商品页选择器.py`

- class 发布商品页选择器
- 从现有 `pages/发布商品页.py` 中提取所有选择器，保留现有值
- 包含：弹窗关闭按钮列表、弹窗关闭文本列表、商品标题输入框、提交并上架按钮、发布成功页URL特征、发布成功页文本、图片容器、图片更换按钮

**每个选择器类的格式统一为：**

```python
class XXX选择器:
    """XXX页面元素选择器"""
    
    元素名称: list[str] = ["选择器1", "选择器2"]
```

每个属性是 `list[str]`，支持多个备选选择器，page 层遍历列表逐个尝试。

**修改文件：**

`pages/基础页.py`

- 顶部 import `from selectors.基础页选择器 import 基础页选择器`
- 所有硬编码选择器替换为引用 `基础页选择器.xxx`
- 动作逻辑保持不变

`pages/登录页.py`

- 同理，import 并引用 `登录页选择器`

`pages/商品列表页.py`

- 同理，import 并引用 `商品列表页选择器`

`pages/发布商品页.py`

- 同理，import 并引用 `发布商品页选择器`
- `关闭所有弹窗` 只用 `发布商品页选择器.弹窗关闭按钮列表` 和 `发布商品页选择器.弹窗关闭文本列表`，不包含任何图片相关选择器

**规则：**

- 只做选择器提取和引用替换，不改任何动作逻辑
- 不改 `tasks/` 目录
- 不改 `browser/` 目录
- 不在选择器文件中放任何非元素定位的内容（不放 URL、不放验证码、不放业务逻辑）
- 页面 URL 常量保留在 page 文件中（如 `商品列表页URL`），不提取到选择器
- 现有已验证的选择器值保留原样，不要改成 TODO

测试验证：`pytest tests/ -x`
