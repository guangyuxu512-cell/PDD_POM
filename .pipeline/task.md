### 问题

当前 `导航并拦截售后列表` 试图用 `asyncio.sleep(2)` 等待导航的默认列表请求完成后再注册拦截器，但 2 秒不够——默认请求的 response 经常在 2 秒之后才到达，仍然被拦截器错误捕获。硬等更长时间也不可靠，因为网络延迟不固定。

### 修复方案：两阶段拦截

不要试图"躲开"默认请求。改为：

1. **第一阶段**：导航后立即注册一个"消耗用"拦截器，专门等默认请求的 response 到达并丢弃
2. **第二阶段**：第一阶段完成后，注册真正的拦截器，然后点击"待商家处理"

### 修改文件

仅修改 `pages/售后页.py`

### 修改方法：`导航并拦截售后列表`

将当前实现替换为：

```python
async def 导航并拦截售后列表(self) -> list[dict]:
    """先导航到列表页，消耗掉默认请求，再通过待商家处理卡片切换触发接口拦截。"""
    await self.导航到售后列表()

    # ===== 第一阶段：消耗默认请求 =====
    # 导航会触发一个默认的售后列表请求（"全部"状态）
    # 我们注册一个临时拦截器，等它的 response 到达后丢弃
    print("[售后页] 等待默认请求完成")
    await self.拦截售后列表API(超时秒=8)
    # 此时默认请求的 response 已经被消耗掉了（或者超时了，都无所谓）
    print("[售后页] 默认请求已消耗，开始真正的拦截")

    # ===== 第二阶段：真正的拦截 =====
    # 注册新的拦截器（全新的 Event 和结果容器）
    拦截任务 = asyncio.create_task(self.拦截售后列表API(超时秒=15))
    await asyncio.sleep(0.1)  # 确保 response 监听器注册完成

    # 点击"待商家处理"触发新的列表请求
    await self.确保待商家处理已选中(强制点击=True)
    结果 = await 拦截任务

    if not 结果:
        print("[售后页] 首次 API 拦截为空，重试")
        拦截任务 = asyncio.create_task(self.拦截售后列表API(超时秒=15))
        await asyncio.sleep(0.1)
        await self.确保待商家处理已选中(强制点击=True)
        结果 = await 拦截任务

    if not 结果:
        print("[售后页] API拦截失败，fallback到JS抓取")

    return 结果
```

### 关键改动说明

1. **第一次 `await self.拦截售后列表API(超时秒=8)`** 是同步等待（不是 `create_task`），它会阻塞直到捕获到默认请求的 response 或超时。捕获到的数据直接丢弃（返回值不使用）。
2. **第一次拦截完成后**，默认请求已经被"消耗"了，此时注册第二个拦截器，它的 Event 和结果容器都是全新的，不会被任何旧 response 影响。
3. 去掉了 `asyncio.sleep(2)` 和 `页面加载延迟()`，不再依赖硬等。
4. 去掉了 `仅待商家处理=True` 参数，因为时序正确后不需要内容过滤。

### 同样修改 `翻页并拦截`

翻页的拦截不需要两阶段（翻页前没有"默认请求"需要消耗），但把 `asyncio.sleep(0)` 改为 `asyncio.sleep(0.1)`，并去掉 `仅待商家处理=True`：

```python
async def 翻页并拦截(self) -> list[dict] | None:
    if not await self._检查有下一页():
        return None

    拦截任务 = asyncio.create_task(self.拦截售后列表API(超时秒=15))
    await asyncio.sleep(0.1)
    翻页成功 = await self.翻页()
    if not 翻页成功:
        拦截任务.cancel()
        return None
    return await 拦截任务
```
